Referencial transparency is key to the functional paradigm, it means the ability to swop out function calls with their resultant values, but it only works when the contraints are followed:

What contraints

  • Must accept input
  • Must have a return value
  • Must only manipulate its inputs

Coming from OOP

Instead of focusing on the nouns and verbs to model the system focus on data transformations

Jargon

Arity
The number of parameters defined in a method signature/declaration
1 parameter: unary, 2 parameter binary, n parameters n-ary
variadic function
A function signature that accepts an indeterminate amount of arguments
Closure
When a function remembers and accesses variables from outside its own scope, even when that function is executed in a different scope
Currying
Taking a multi parameter function and breaking it down into a chain of functions that each takes a single argument and returns another function to accept the next argument. (unwinds a single higher-arity function into a series of chained unary functions.)
Partial Application
Reduction of a functions arity
Applicative
functor
Referencial transparency
The ability to swop all function calls in code with the return value fro making that function call without breaking it
